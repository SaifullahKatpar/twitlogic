SPARQL-based Twitter filters
    For the International Semantic Web Conference 2010

    A demo by Joshua Shinavier
Below is an embeddable widget which, like Twitter's search widget, provides a live view of the Twitter stream. However, this one allows you to create much "smarter" streams of social data, using SPARQL. Why is this cool? Well, the more data that can be gotten out of a tweet, the more sophisticated real-time filters you can create. Imagine a stream of tweets by people with low Erdös numbers, or about upcoming albums by bands you like. Rather than developing an entirely new Twitter application, just write a new SPARQL query. The example below is connected to a SPARQL endpoint which contains the Semantic Web Conference Corpus, a DBpedia dump, special-purpose data for ISWC 2010, and a real time stream of tweets about the conference. Based on the authors and #hashtags of tweets, we can begin to construct some interesting filters.
To use this demo:
  • Select a filter from this list:
  • Watch the tweets appear. Any subsequent tweets will appear as soon as they arrive, through Twitter's Streaming API.
  • You can easily customize this widget for your own application, changing the query, the SPARQL endpoint, the style of the widget, etc. Have a look at the source code.
To see the SPARQL query for each filter, click on the "SPARQL powered" link at the lower right corner of the widget.

Disclaimer: this widget is very new, so there may be bugs. I believe it works in Firefox, Chrome, Safari and Opera. It does not yet work in Internet Explorer. If you encounter any other issues, please let me know.
