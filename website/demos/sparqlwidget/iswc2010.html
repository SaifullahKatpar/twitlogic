SPARQL-based Twitter filters
For the International Semantic Web Conference 2010

Below is a widget which, like Twitter's search widget, provides a live view of the Twitter stream. However, this one allows you to create much "smarter" streams of social data, using SPARQL. So why is this cool? Well, the more data that can be gotten out of a tweet, and the more background knowledge you can combine it with, the more sophisticated real-time filters you can create. Imagine a stream of tweets by people with low Erdos numbers, or about new albums by bands you like. The example below is connected to a SPARQL endpoint which contains the Semantic Web Conference Corpus, a DBpedia dump, and special-purpose data for ISWC 2010. Combined with the hashtags found in tweets, we can begin to construct some interesting filters.
